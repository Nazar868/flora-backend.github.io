# Flora — Floral Artistry

Семантична, адаптивна HTML/CSS-верстка сторінки квіткової майстерні **Flora**, виконана за критеріями 1-го скоупу проєкту (заняття 1–12).

## Демо

Після деплою додайте посилання на GitHub Pages сюди та в секцію **About** репозиторію.

## Структура проєкту

```
├── index.html
├── css/
│   ├── modern-normalize.css   # скидання стилів
│   └── styles.css             # усі стилі проєкту, CSS-змінні, адаптив
├── images/
│   ├── icons.svg               # SVG-спрайт іконок (<use>)
│   └── *.jpg                   # оптимізовані фото (варіанти @1x / @2x)
├── js/
│   └── main.js                 # мобільне меню (is-open), Show More, AOS-анімації
├── favicon.ico
└── favicon-32.png
```

## Що реалізовано

- Семантична розмітка: `header`, `main`, `footer`, один `nav`, один `h1`, `h2` для секцій, списки в `ul`.
- Шрифти Fraunces + Work Sans з Google Fonts (одне посилання), `font-family` заданий лише на `body`.
- Кольори винесені в CSS-змінні (`:root`), підключений modern-normalize, `box-sizing: border-box`.
- Контейнер `.container` (1336px), відступи через `margin`/`padding`, flexbox — там, де потрібен, без фіксованих висот.
- Mobile-first адаптив на 375 / 768 / 1440 px, без горизонтальної прокрутки від 320 px.
- SVG-іконки зібрані у спрайт `images/icons.svg`, підключені через `<use>`.
- Багатошаровий фон (градієнт + зображення) у hero-секції.
- Переходи 250ms `cubic-bezier(0.4, 0, 0.2, 1)` на інтерактивних елементах, явні властивості (без `all`).
- Підключена бібліотека анімацій AOS (`data-aos` атрибути).
- Мобільне меню на всю висоту в’юпорту, приховане за замовчуванням, відкривається класом `is-open`.
- Без `!important`, без console-помилок/логів.

## Деплой на GitHub Pages

1. Перейменуйте репозиторій у формат `UMT-markup-practice_P1-ПІБ`.
2. Запуште вміст цієї папки в гілку `main`.
3. У `Settings → Pages` виберіть джерело `main` / `root` і збережіть.
4. Додайте отримане посилання GitHub Pages у секцію **About** репозиторію.
5. Видаліть усі робочі гілки, окрім `main` та `gh-pages`.
